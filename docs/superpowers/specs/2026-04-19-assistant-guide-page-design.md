# AI 智能导览页界面设计

## Scope

本设计只覆盖 `assistant-app` 的 AI 智能导览页：布局、视觉层次、聊天区、快捷问题和移动端适配。主站入口、DeepSeek 接口、知识库内容和部署结构沿用现有方案。

## Direction

页面采用“沉浸式导览台”方向。背景保留黄鹤楼水墨画，界面不做普通网页卡片堆叠，而像展厅里的半透明讲解台。左侧负责导览入口与快捷问题，右侧负责对话和结构化内容展示。

参考图中的关键特征需要保留：

- 左侧固定导览面板，包含返回按钮、英文标识、标题、说明、五个快捷问题和红色印章。
- 右侧为主操作区，顶部有斜角标签，显示 “AI 智能导览 / 黄鹤楼数字展厅小助手”。
- 聊天区能展示用户问题、AI 回复，以及一个“黄鹤楼层分布”内容展板。
- 输入框保持在聊天区底部附近，视觉上轻、稳定、易发现。
- 整体颜色以水墨灰、宣纸白、深墨色和少量朱红为主。

## Desktop Layout

桌面端使用左右分区：

- 页面全屏铺满视口，不出现独立外框。
- 背景图全屏覆盖，黄鹤楼主体偏中上，保证右侧聊天区仍有视觉焦点。
- 左侧导览面板宽度约 320px，距离左侧和顶部留出展厅感边距。
- 右侧聊天区从左侧面板右侧开始，宽度随视口扩展，底部留出呼吸空间。

右侧聊天区内部：

- 顶部标签置于聊天区外沿上方，使用浅色半透明底和斜角轮廓。
- 消息区顶部先展示用户问题，再展示 AI 回复。
- AI 回复下方放置“黄鹤楼层分布”展板，包含标题、楼层清单和一张小型水墨图。
- 上下文计数保留但弱化，放在右上角。
- 输入框位于底部，宽度受容器约束，避免贴边。

## Components

`AssistantGuidePanel`

- 返回按钮使用左箭头和“返回数字展厅”。
- 快捷问题保持当前五个问题，但视觉上统一为固定高度按钮。
- 印章放在面板右下角。

`AssistantChat`

- 使用现有 Chatscope 消息组件承载真实消息。
- 在消息列表下方增加一个静态导览内容展板，用于首屏展示楼层结构。
- 当前版本不新增复杂路由或状态机。

`FloorGuideCard`

- 标题：“黄鹤楼层分布”。
- 左侧展示五层清单。
- 右侧展示黄鹤楼水墨图裁切图，作为内容辅助。
- 只作为首屏导览提示，不替代 AI 回复。

## Mobile Layout

移动端改为纵向布局：

- 顶部为压缩导览面板，包含返回按钮、标题、说明和横向滚动快捷问题。
- 聊天区从面板下方开始，占据剩余屏幕。
- 消息头像缩小，消息最大宽度改为容器宽度。
- “黄鹤楼层分布”展板改为单列，图片放在清单下方。
- 输入框固定在聊天区底部视觉区域内，避免被快捷问题挤压。

## Accessibility And Stability

- 所有按钮保持原生 `button`。
- 装饰图使用空 `alt` 和 `aria-hidden`。
- 不使用随视口宽度变化的正文号，避免窄屏文字溢出。
- 固定头像、按钮和输入框尺寸，减少消息变化带来的布局跳动。

## Verification

完成后运行：

- `npm --prefix assistant-app run build`
- `npm test`

如需视觉确认，启动本地服务并截取桌面与移动端页面，确认：

- 背景图可见且不遮挡文字。
- 左侧导览面板和右侧聊天区接近参考图。
- 楼层分布展板可读。
- 移动端内容不溢出，输入框可见。
