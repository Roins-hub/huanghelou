# 交互网页部署说明

这个项目是一个纯前端静态站点，核心文件是：

- `index.html`
- `script.js`
- `style.css`
- `lou/` 模型与贴图目录

它依赖：

- 本地静态服务器访问模型文件
- 外部 CDN 资源：Three.js、MediaPipe、Google Fonts
- 摄像头权限（手势识别）

## 方案一：局域网内让其他设备访问

适合：

- 你的电脑和手机/平板在同一个 Wi-Fi
- 先演示页面，不一定强依赖手机摄像头手势

启动方法：

```powershell
cd D:\3Dmoxing
.\start-lan-server.ps1
```

然后在其他设备打开：

```text
http://10.28.110.1:8000/
```

注意：

- Windows 防火墙第一次可能会弹窗，记得允许 `Python` 访问专用网络。
- 如果能打开页面但摄像头不能授权，这是浏览器安全策略导致的，不一定是代码问题。

## 方案二：部署到 HTTPS 静态托管（推荐）

适合：

- 需要手机正常调用摄像头
- 需要外网访问
- 需要更稳定地展示或分享链接

推荐平台：

- Netlify
- Cloudflare Pages
- Vercel
- GitHub Pages（可用，但交互类站点我更推荐前 3 个）

### 最省事的部署方式

1. 把整个 `D:\3Dmoxing` 目录上传到 GitHub 仓库。
2. 在 Netlify 或 Cloudflare Pages 里新建站点并连接仓库。
3. 构建设置保持为空或按下面填写：

```text
Build command: 留空
Publish directory: .
```

4. 部署完成后会得到一个 `https://` 链接。

### 部署时必须包含的文件

- `index.html`
- `script.js`
- `style.css`
- `lou/huanghelou.obj`
- `lou/material.mtl`
- `lou/*.png`

如果漏掉 `lou/` 目录，3D 模型会加载失败。

## 已知限制

1. 页面依赖公网 CDN；如果部署环境或访问设备无法连接这些 CDN，Three.js 或 MediaPipe 会加载失败。
2. 摄像头能力依赖浏览器支持与 HTTPS 安全上下文。
3. 如果你只是在电脑本机访问，`http://localhost:8000/` 一般就可以正常调试。

## 当前建议

如果你现在只是想让别的设备马上看到页面：

```powershell
cd D:\3Dmoxing
.\start-lan-server.ps1
```

如果你想让手机也稳定使用手势交互：

- 把项目部署到 Netlify 或 Cloudflare Pages
- 使用平台分配的 `https://` 地址访问
